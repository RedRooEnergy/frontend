# Service Partner Events — Contract Definition

This document defines Service Partner–originated audit events.

Events are observational only and must not trigger state changes.

Examples (non-exhaustive):
- SERVICE_PARTNER_TASK_VIEWED
- SERVICE_PARTNER_ASSIGNMENT_ACKNOWLEDGED
- SERVICE_PARTNER_EVIDENCE_UPLOAD_STARTED
- SERVICE_PARTNER_EVIDENCE_UPLOAD_COMPLETED
- SERVICE_PARTNER_TASK_COMPLETION_ACKNOWLEDGED

All events are audit-only.
