# Admin Oversight and Approvals
Version: v1.0 (LOCKED)  
Scope: Governance-only; no implementation

## Visibility
- Admins can view payments, refunds, settlements, disputes, compliance status.
- Dashboards must surface escrow status, review flags, and overrides applied.

## Override Limitations
- Overrides (payment review clearance, settlement override, refund override) require:
  - Recorded approval ID
  - Approver role and timestamp
  - Linked rationale

## Approval Authorities
- Payment review clearance: finance/compliance admin.
- Settlement override: senior admin only.
- Staged payment waiver: finance + compliance dual approval.

## Logging & Audit
- All admin actions must emit audit events with order ID, action, approver, timestamp.
- No silent state changes; webhook/app changes must be immutable and auditable.
