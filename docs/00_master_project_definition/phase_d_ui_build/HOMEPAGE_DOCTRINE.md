# Homepage Doctrine

Purpose:
Establish trust through governance, enforcement, and transparency
before presenting features or conversion paths.

Mandatory Sections:
1. What This Platform Is
2. Why Governance Matters
3. How Enforcement Protects Buyers & Suppliers
4. Extension-Based Capabilities (plain language)
5. Role-Specific Calls to Action

Mandatory Claims (Must Be True):
- Prices are locked at checkout
- Compliance is verified before shipment
- Audit trails are immutable
- No bypass of enforced rules

Forbidden:
- Claims of flexibility that contradict enforcement
- Feature promises not enforced by backend
- Speed claims that undermine compliance

Rule:
Trust > Enforcement > Features > Conversion
